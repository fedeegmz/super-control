Registrar un usuario

  • Method: POST
  • Endpoint: /users/signup
  • Body parameters (JSON):
    • username: entre 4 y 15 caracteres
    • name: entre 3 y 20 caracteres
    • lastname: entre 3 y 20 caracteres
    • email
    • birth_date: (opcional) en formato AAAA/MM/DD
    • password: entre 8 y 64 caracteres
  • Return: usuario creado
En el caso de que el username ya este registrado se retorna un error 409.
Si hubo un problema al registrarse el usuario tambien se retorna un error 409.

Obtener todos los usuarios

  • Method: GET
  • Endpoint: users/
  • Return: lista de usuarios
En el caso de que ocurra un inconveniente con la base de datos se retorna un error 409.

Obtener un usuario

  • Method: GET
  • Endpoint: /users/{username}
  • Path parameter: username
  • Return: usuario
Si el username no esta registrado se retorna un error 404.
Y si hubo un problema con la base de datos se retorna un error 409.

Actualizar un usuario

  • Method: PATCH
  • Endpoint: /users/{username}
  • Path parameter: username
  • Body parameters: campos a actualizar (Ej. {"name": "Wade"})
  • Return: usuario actualizado
Si el username no esta registrado se retorna un error 404.
Y si hubo un problema con la base de datos, o el usuario fue eliminado, se retorna un error 409.

Eliminar un usuario

  • Method: DELETE
  • Endpoint: /users/{username}
  • Path parameter: username
  • Return: usuario eliminado
Si el username no esta registrado se retorna un error 404.
Y si hubo un problema con la base de datos, o el usuario ya fue eliminado, se retorna un error 409.
